# 4フレームワーク メリット・デメリット比較

対象：ChatDev / MetaGPT / OpenHands / AIKAISYA（本プロジェクト）

---

## 一覧比較表

| 項目 | ChatDev | MetaGPT | OpenHands | AIKAISYA |
|---|---|---|---|---|
| コンセプト | 仮想ソフトウェア会社 | SOP駆動の多役職チーム | AIが実際に手を動かす | 仮想API開発会社 |
| 主な出力 | ソフトウェア全般 | コード＋仕様書＋ドキュメント | コード実行・ファイル操作 | API |
| カスタマイズ性 | JSONで会社構造を定義 | 役職・SOPを設定可 | エージェントSDKで自由設計 | 部門・方針をYAMLで定義 |
| 実行能力 | コード生成のみ | コード生成のみ | 実際に実行・操作 | （設計中） |
| 特徴的な機能 | 会社の自動進化（v2） | 1行→完全成果物 | Docker サンドボックス | 方針蓄積・自己進化・API特化 |
| 学習コスト | 低 | 中 | 中 | 低（自分で設計） |
| ライセンス | Apache 2.0 | MIT | MIT | 独自 |

---

## ChatDev

### メリット
- 仮想会社の構造がシンプルで理解しやすい
- 3つのJSONファイルで会社を丸ごとカスタマイズできる
- v2.0でノーコード化され、汎用性が大幅に向上
- 組織の自己進化（新部門の自動追加）機能がある

### デメリット
- ソフトウェア開発に特化しており、API開発への転用に工夫が必要
- v1（仮想会社モデル）とv2（汎用プラットフォーム）でコンセプトが大きく変わり、ドキュメントが分散
- 実際のコード実行環境（サンドボックス）が弱い

---

## MetaGPT

### メリット
- SOP（標準業務手順）という概念が強力で、品質が安定しやすい
- 1行の要件から仕様書・コード・ドキュメントまで一気通貫で生成
- 役職の分担が明確で、本プロジェクトの部門構造と最も親和性が高い
- pip一発でインストールできる手軽さ

### デメリット
- 役職が固定的（プロダクトマネージャー・アーキテクト・エンジニア等）で、API特化の部門を追加しにくい
- コード実行は生成のみで、実際にデプロイ・動作確認はできない
- Python 3.12非対応（3.9〜3.11のみ）

---

## OpenHands

### メリット
- AIが実際にコードを実行・デバッグ・修正までできる（生成だけでない）
- DockerサンドボックスによりPC環境を汚さず安全に実行できる
- CLI・GUI・Cloud・Enterpriseと導入形態が豊富
- GitHub/GitLab/Slack/Jira連携など外部ツールとの統合が充実
- 大手企業（Amazon・Google・Netflix等）の実績がある

### デメリット
- 「会社組織」の概念が薄く、部門間の連携・役割分担の設計が自分任せ
- エンタープライズ機能は有料
- 自由度が高い分、使いこなすまでの学習コストがかかる

---

## AIKAISYA（本プロジェクト）

### メリット
- **API開発に特化**した唯一のフレームワーク（他は汎用ソフトウェア開発向け）
- トレンド確認係・使用量確認係を全部門に配置する**コスト意識と技術鮮度の仕組み**が独自
- 方針蓄積部門・方針共有部門による**組織の記憶と学習**の仕組みがある
- 新規部門立案・判定部門による**自己進化**の仕組みがある
- 業績発表・社外アピール部門など**ビジネスサイドまで一気通貫**している
- 自分で設計するため、要件に完全に合わせられる

### デメリット
- **まだ実装されていない**（現時点は構想のみ）
- 部門数が多く、全部を動かすには相当な実装量が必要
- 既存フレームワークのように即座に使えるエコシステムがない
- 実行環境（サンドボックス）の設計がこれから

---

## まとめ：本プロジェクトへの活用方針

| フレームワーク | 活用できる部分 |
|---|---|
| ChatDev | 3つのJSONで会社を定義する設計思想 → `config/policy.yaml` の参考 |
| MetaGPT | SOP＝共通方針の概念、役職→エージェントの対応設計 |
| OpenHands | Dockerサンドボックスによる実行環境、製造工程の実装参考 |

**推奨戦略：**
MetaGPTをベースに役職をAPI会社向けにカスタマイズし、
OpenHandsの実行環境を組み合わせることで、
最短距離でAIKAISYAのPhase 1〜3を実現できる。
