# ChatDev 概要

参照：https://github.com/OpenBMB/ChatDev

---

## ChatDevとは

LLM（大規模言語モデル）を使った**仮想ソフトウェア会社**。
複数のAIエージェントがそれぞれ役職を持ち、会話（チャット）しながら協働してソフトウェアを自動生成するフレームワーク。

「アイデアを伝えるだけで、設計→コーディング→テスト→ドキュメントまでを自動でこなす」のがコンセプト。

---

## できること

| 機能 | 内容 |
|---|---|
| ソフトウェア自動生成 | テキストの指示からコードを自動生成 |
| データ可視化 | 大規模データセットから高品質チャートを自動生成 |
| ゲーム開発 | マルチエージェントで協調してゲームを開発 |
| 3D生成 | Blenderと連携して3Dモデルを自動構築 |
| 動画生成 | 教育コンテンツの自動制作 |
| 深型リサーチ | 複数エージェントによる情報収集と分析 |

## 基本構造（v1.0：仮想ソフトウェア会社モデル）

### 役職（エージェント）

| 役職 | 役割 |
|---|---|
| CEO（最高経営責任者） | プロジェクト全体の意思決定 |
| CPO（最高製品責任者） | 製品要件・仕様の管理 |
| CTO（最高技術責任者） | 技術方針・アーキテクチャの決定 |
| プログラマー | コードの実装 |
| レビュアー | コードレビュー |
| テスター | テスト・品質確認 |
| アートデザイナー | UI・ビジュアルデザイン |

### 開発フェーズ

```
設計（Design）
  └─▶ コーディング（Coding）
        └─▶ テスト（Testing）
              └─▶ ドキュメント作成（Documentation）
```

各フェーズで対応するエージェントが「セミナー形式の会話」を行い、バトンを渡していく。

### カスタマイズの仕組み（3つのJSONファイル）

| ファイル | 役割 |
|---|---|
| `ChatChain.json` | 開発プロセス全体のフロー定義 |
| `Phase.json` | 各フェーズの詳細設定 |
| `Role.json` | エージェントの役割・プロンプト定義 |

この3ファイルを編集するだけで、独自の「ChatDev会社」を構築できる。

## アーキテクチャ（v2.0：DevAll）

v2.0ではコーディング特化から「**ゼロコード多エージェント・オーケストレーションプラットフォーム**」に進化。

| 層 | 内容 |
|---|---|
| エージェント層 | 専門性の異なる知的エージェント群 |
| ワークフロー層 | ドラッグ&ドロップでエージェント連携を設計 |
| 実行層 | タスク実行とログ監視 |

**使用技術：**
- バックエンド：FastAPI（Python 3.12+）
- フロントエンド：Vue 3 + Vite
- 実行：Docker Compose対応

## 本プロジェクトとの関連

| ChatDev | 本プロジェクト（仮想API作成会社） |
|---|---|
| 仮想ソフトウェア会社 | 仮想API開発会社 |
| CEO / CTO / プログラマーなど | トレンド確認部門 / セキュリティー部門 / 製造工程管理部門など |
| ChatChain（フロー定義） | workflow.py（部門間連携フロー） |
| Role.json（役割定義） | base_agent.py + policy.yaml |
| 設計→コーディング→テスト→ドキュメント | トレンド確認→設計→製造→検証→発信 |

ChatDevのアーキテクチャは、本プロジェクトの**実装参考モデル**として活用できる。
特に「3つのJSONで会社をカスタマイズ」する設計思想は、`config/policy.yaml` の方針管理と共通する。
