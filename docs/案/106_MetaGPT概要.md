# MetaGPT 概要

参照：https://github.com/geekan/MetaGPT

---

## MetaGPTとは

複数のAIエージェントがソフトウェア会社の役職を担い、**SOP（標準業務手順）に沿って協働**するマルチエージェントフレームワーク。

核心思想：**「Code = SOP(Team)」**
— ソフトウェア企業の実際の業務フローをLLMチームで再現する。

1行の要件を入力するだけで、ユーザーストーリー・競合分析・仕様書・データ構造・API設計・ドキュメントまで自動生成する。

---

## できること

| 機能 | 内容 |
|---|---|
| ソフトウェア自動生成 | 1行の要件から設計〜コードまで一気通貫 |
| ユーザーストーリー生成 | 要件を自動でユーザー視点に変換 |
| 競合分析 | 市場・競合の自動リサーチ |
| API仕様書生成 | OpenAPI形式の仕様書を自動作成 |
| データ構造設計 | DB・モデル設計の自動化 |
| データ分析 | 数値データの分析・可視化 |
| リサーチ | 特定テーマの調査・レポート作成 |

## 基本構造

### 役職（エージェント）

| 役職 | 役割 |
|---|---|
| プロダクトマネージャー | 要件分析・戦略立案・ユーザーストーリー作成 |
| アーキテクト | システム設計・技術方針決定 |
| プロジェクトマネージャー | スケジュール管理・進捗監視 |
| エンジニア | 実装・開発作業 |

### ワークフロー（SOPに基づく流れ）

```
1行の要件（自然言語）
  └─▶ プロダクトマネージャー（要件分析・ユーザーストーリー）
        └─▶ アーキテクト（システム設計・API設計）
              └─▶ プロジェクトマネージャー（タスク分解・スケジュール）
                    └─▶ エンジニア（コード実装）
                          └─▶ 出力（コード＋仕様書＋ドキュメント）
```

### SOPの概念

各役職が「この役職ならこう動く」というSOPに従って動作。
人間の会社の業務フローをそのままLLMに適用するのが設計思想。

## 本プロジェクトとの関連

| MetaGPT | 本プロジェクト（仮想API作成会社） |
|---|---|
| プロダクトマネージャー | トレンド確認部門・会社将来方針決定課 |
| アーキテクト | 新技術調整部門・ドキュメント作成部門 |
| プロジェクトマネージャー | 製造工程管理部門 |
| エンジニア | 製造工程分解部門 |
| SOP | 共通方針部門・方針蓄積部門 |

MetaGPTの「SOP＝会社の憲法」という考え方は、本プロジェクトの**共通方針部門**の設計と完全に一致する。最も参考にすべきフレームワーク。
